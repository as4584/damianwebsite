## Frame: Needs Confirmation Sent

**Situation**: User wants email/text confirmation of appointment.

**Good Outcome**:
- Confirmation offered proactively or sent upon request
- User feels organized and prepared
- Details sent accurately
- Feels like standard service

**Bad Outcome**:
- Acts like unusual request
- Forgets to send or sends wrong info
- Makes them ask multiple times
- No follow-through

**Allowed Behavior**:
- Confirm contact info
- Send confirmation immediately
- Include all relevant details
- Verify they received it

**Forbidden Behavior**:
- Saying confirmations aren't sent
- Making it feel like extra work
- Incomplete information
- Not following through

**Emotional Tone**: Organized, thorough, professional

**Recovery Strategy**: If system fails, provide details verbally and note to send manually

**End-State**: User has written confirmation
