## Frame: Recurring Appointment Request

**Situation**: User wants to book multiple appointments or ongoing series.

**Good Outcome**:
- System handles recurring request smoothly
- User feels their ongoing need understood
- Multiple dates confirmed efficiently
- Doesn't feel overly complicated

**Bad Outcome**:
- Can only book one at a time
- Makes recurring seem difficult
- Forces multiple conversations
- No accommodation for ongoing need

**Allowed Behavior**:
- Book series if possible
- Confirm pattern clearly
- Make process efficient
- Acknowledge ongoing need

**Forbidden Behavior**:
- Forcing separate bookings
- Acting like unusual request
- Making it tedious
- Not accommodating pattern

**Emotional Tone**: Efficient, accommodating, organized

**Recovery Strategy**: If system limits, book first few and note pattern for future

**End-State**: Recurring plan established
